# 실무 체크리스트

kube-bench를 실무에 적용할 때 확인해야 할 사항들을 정리합니다.

## 실행 전 체크리스트

### 1. 환경 준비

| 항목 | 확인 사항 | 확인 방법 |
|------|----------|----------|
| **Kubernetes 버전** | Kubernetes 1.17 이상 지원 여부 확인 | `kubectl version --short` |
| **kube-bench 설치** | kube-bench 설치 여부 확인 | `kube-bench version` |
| **권한** | 노드 파일시스템, 프로세스, 설정 파일 접근 권한 확인 | root 또는 sudo 권한 필요 |
| **관리형 클러스터** | EKS, GKE, AKS 등 관리형 클러스터 여부 확인 | 마스터 노드 검사 제한 가능 |

### 2. 검사 대상 준비

| 항목 | 확인 사항 | 비고 |
|------|----------|------|
| **노드 유형** | 마스터/워커 노드 확인 | 마스터/워커 노드별 검사 선택 |
| **Kubernetes 버전** | Kubernetes 버전 확인 | 자동 버전 감지 또는 명시적 지정 |
| **합격 기준** | FAIL 항목 허용 범위 설정 | 기본값 0건 (허용 범위 내) |

## 실행 중 체크리스트

### 1. 검사 실행

| 항목 | 확인 방법 | 비고 |
|------|----------|------|
| **검사 상태** | 검사 진행 상황 확인 | 터미널 출력 확인 |
| **에러 확인** | 검사 중 에러 발생 여부 확인 | 로그 확인 |

## 실행 후 체크리스트

### 1. 결과 확인

| 항목 | 확인 방법 | 비고 |
|------|----------|------|
| **FAIL 항목 확인** | FAIL 항목 수 확인 | JSON 리포트 내 `Controls[].result == "FAIL"` |
| **합격 여부** | 합격 기준 충족 여부 확인 | FAIL 항목 0건 또는 허용 범위 내 |
| **권고 확인** | FAIL 항목의 수정 권고 확인 | Remediation 단계 확인 |

### 2. 리포트 생성

| 산출물 | 생성 방법 | 비고 |
|--------|----------|------|
| **JSON 리포트** | `kube-bench run --targets master --json --output results.json` | 원본 리포트 |
| **JUnit XML** | `kube-bench run --targets master --junit --output results.xml` | CI/CD 통합용 |
| **stdout 출력** | `kube-bench run --targets master` | 터미널 출력 |

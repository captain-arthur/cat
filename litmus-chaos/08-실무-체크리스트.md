# 실무 체크리스트

LitmusChaos를 실무에 적용할 때 확인해야 할 사항들을 정리합니다.

## 실행 전 체크리스트

### 1. 환경 준비

| 항목 | 확인 사항 | 확인 방법 |
|------|----------|----------|
| **Kubernetes 버전** | Kubernetes 1.17 이상 지원 여부 확인 | `kubectl version --short` |
| **LitmusChaos 설치** | LitmusChaos 설치 여부 확인 | `kubectl get pods -n litmus` |
| **CRD 확인** | ChaosExperiment, ChaosEngine CRD 확인 | `kubectl get crd | grep litmuschaos` |
| **권한** | Kubernetes 리소스 조작 권한 확인 | `kubectl auth can-i create chaosengine --all-namespaces` |
| **Persistent Volume** | ChaosCenter 저장소용 PV 확인 | `kubectl get pvc -n litmus` |

### 2. 실험 CRD 준비

| 항목 | 확인 사항 | 비고 |
|------|----------|------|
| **ChaosEngine 문법 검증** | CRD YAML 문법 검증 | `kubectl apply --dry-run=client -f chaosengine.yaml` |
| **ChaosHub 템플릿** | ChaosHub에서 실험 템플릿 가져오기 | ChaosCenter UI 또는 CLI 사용 |
| **Probes 설정** | Probe 정의 및 설정 확인 | Steady State 검증 확인 |

## 실행 중 체크리스트

### 1. 상태 모니터링

| 항목 | 확인 방법 | 비고 |
|------|----------|------|
| **ChaosEngine 상태** | `kubectl get chaosengine -n default` | 정기적으로 확인 |
| **ChaosResult 상태** | `kubectl get chaosresult -n default` | 실험 결과 확인 |
| **Probe 결과** | `kubectl describe chaosresult <name> -n <namespace>` | Probe 검증 결과 확인 |

## 실행 후 체크리스트

### 1. 결과 확인

| 항목 | 확인 방법 | 비고 |
|------|----------|------|
| **ChaosResult verdict** | `kubectl get chaosresult -n default -o jsonpath='{.status.experimentStatus.verdict}'` | verdict: Pass 확인 |
| **Probe 검증 결과** | `kubectl describe chaosresult <name> -n <namespace>` | Probe 검증 통과 여부 |

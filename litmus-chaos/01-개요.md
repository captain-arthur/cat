# LitmusChaos 개요

## LitmusChaos란?

LitmusChaos는 쿠버네티스 클러스터 및 애플리케이션의 **복원력(Resilience)과 장애 대응 능력**을 실험을 통해 검증하는 Kubernetes 네이티브 Chaos Engineering 플랫폼입니다.

| 속성 | 설명 |
|------|------|
| **목적** | 시스템이 장애 상황에서도 정상 동작을 유지하는지 실험을 통해 검증 |
| **주체** | CNCF Incubating 프로젝트 (Apache 2.0 라이선스) |
| **특징** | Kubernetes CRD 기반, ChaosCenter UI 제공, ChaosHub 템플릿, Probes 검증 |
| **언어** | Go, TypeScript |
| **지원 환경** | Kubernetes (네이티브 통합) |

## 주요 기능

| 기능 | 상세 설명 |
|------|----------|
| **ChaosHub** | 실험 템플릿 저장소로 다양한 실험 템플릿 제공 |
| **ChaosCenter** | 웹 기반 UI로 실험 생성, 모니터링, 관리 |
| **Probes** | Steady State 검증을 위한 다양한 Probe 타입 지원 |
| **Chaos Workflow** | 여러 실험을 직렬/병렬로 구성하여 복합 실험 실행 |
| **GitOps 지원** | Git 기반 실험 관리 및 자동화 |
| **다중 환경 지원** | 여러 클러스터 및 환경에서 실험 실행 |

## 주요 사용 사례

| 사용 사례 | 설명 |
|----------|------|
| **Pod 복원력 검증** | "Pod가 죽었을 때 자동으로 복구되는가?" 검증 |
| **네트워크 장애 테스트** | 네트워크 지연, 패킷 손실 등 시뮬레이션 |
| **인프라 장애 테스트** | 노드 드레인, 디스크 오류 등 시뮬레이션 |
| **리소스 스트레스 테스트** | CPU/메모리 부하 주입으로 스트레스 테스트 |
| **인수 테스트** | 클러스터 배포 후 장애 대응 능력 확인 |

## 장점

| 장점 | 설명 |
|------|------|
| **ChaosHub** | 실험 템플릿 저장소로 재사용 가능한 실험 공유 |
| **Probes** | Steady State 검증을 위한 강력한 Probe 기능 |
| **ChaosCenter** | 웹 UI로 실험 관리 및 모니터링 용이 |
| **GitOps 지원** | Git 기반 실험 관리 및 자동화 |
| **다중 환경** | 여러 클러스터 및 환경에서 실험 실행 가능 |

## 제약 사항

| 제약 | 설명 |
|------|------|
| **Kubernetes 전용** | Kubernetes 환경에서만 실행 가능 |
| **Persistent Volume** | ChaosCenter 저장소용 PV 필요 (기본 20GB) |
| **리소스 요구** | ChaosCenter 운영을 위한 충분한 리소스 필요 |

## Sonobuoy, ClusterLoader2, kube-burner, Chaos Toolkit, Chaos Mesh와의 관계

| 도구 | 목적 | 특징 |
|------|------|------|
| **Sonobuoy** | Conformance 테스트 (기본 자격 검증) | 기능 정상 동작 여부 검증 |
| **ClusterLoader2** | 성능/스케일 테스트 | 내장 measurement, 구조화된 리포팅 |
| **kube-burner** | 성능/스케일 테스트 | Prometheus 중심 관측/리포팅 |
| **Chaos Toolkit** | 복원력/장애 대응 테스트 | Python 기반, Steady-State Hypothesis |
| **Chaos Mesh** | 복원력/장애 대응 테스트 | Kubernetes 네이티브, CRD 기반, Dashboard |
| **LitmusChaos** | 복원력/장애 대응 테스트 | Kubernetes 네이티브, ChaosHub, Probes, ChaosCenter |

## CAT 관점에서의 역할

LitmusChaos는 **쿠버네티스 클러스터 인수테스트(CAT)**의 **"복원력/장애 대응(Resilience/Fault Tolerance)"** 카테고리에 해당하는 핵심 도구입니다.

### 검증 질문(Assertion) 예시

- "Pod가 죽었을 때 자동으로 재생성되는가?"
- "네트워크 지연 상황에서도 서비스가 정상 응답하는가?"
- "노드 장애 시 워크로드가 다른 노드로 이동하는가?"

자세한 내용은 [06-CAT-활용.md](./06-CAT-활용.md) 참조.

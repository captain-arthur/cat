# kube-burner 개요

## kube-burner란?

kube-burner는 쿠버네티스 클러스터의 **성능 및 스케일 테스트**를 수행하는 도구입니다.

| 속성 | 설명 |
|------|------|
| **목적** | 클러스터 성능 수준, 수용량, 확장성 검증 |
| **접근 방식** | 대량 리소스 생성/삭제 + Prometheus 메트릭 수집 |
| **위치** | CNCF Sandbox 프로젝트 |
| **주체** | Red Hat 주도 개발 (오픈소스) |

## 주요 기능

| 기능 | 상세 설명 |
|------|----------|
| **대량 리소스 생성** | 대규모 Pod, Namespace, Deployment 등을 생성하여 부하 테스트 |
| **성능 지표 수집** | Prometheus와 연계하여 Pod latency, API latency 등 측정 |
| **다양한 워크로드** | cluster-density, node-density, networkpolicy 등 사전 정의된 시나리오 제공 |
| **임계치 검증** | p50, p95, p99 등 통계 및 threshold 기반 Pass/Fail 판단 |
| **결과 분석** | 측정 결과를 시각화 및 인덱싱하여 분석 가능 |
| **OpenShift 지원** | OpenShift 환경 특화 워크로드 (`kube-burner-ocp`) 제공 |

## 주요 사용 사례

| 사용 사례 | 설명 |
|----------|------|
| **성능 검증** | 클러스터가 요구 성능 수준을 만족하는지 확인 |
| **수용량 테스트** | 최대 Pod/Node 수용량 확인 |
| **확장성 검증** | 스케일 업/다운 시 성능 변화 확인 |
| **병목 지점 발견** | API Server, etcd 등 컨트롤 플레인 병목 확인 |
| **SLA 검증** | Pod Ready 시간, API latency 등 SLA 기준 충족 확인 |

## 장점

| 장점 | 설명 |
|------|------|
| **실제 부하 시뮬레이션** | 실제 워크로드와 유사한 부하 생성 |
| **정량적 측정** | 수치 기반 성능 평가 (latency, throughput) |
| **표준화된 시나리오** | 검증된 워크로드 시나리오 제공 |
| **Prometheus 통합** | 기존 모니터링과 연계하여 측정 |
| **임계치 기반 판단** | 명확한 Pass/Fail 기준 설정 가능 |

## 제약 사항

| 제약 | 설명 |
|------|------|
| **Prometheus 필요** | 메트릭 수집을 위해 Prometheus 필요 |
| **리소스 사용** | 대량 리소스 생성으로 인한 클러스터 부하 |
| **실행 시간** | 대규모 테스트는 상당한 시간 소요 |
| **환경 준비** | 테스트 환경 사전 준비 필요 |

## CAT 관점에서의 역할

kube-burner는 **쿠버네티스 클러스터 인수테스트(CAT)**의 **"성능/수용량(Performance/Scale)"** 카테고리에 해당하는 핵심 도구입니다.

### 검증 질문(Assertion) 예시

- "클러스터가 요구 성능 수준(p95 API latency < X ms)을 만족하는가?"
- "N 노드에서 X Pod 생성 시 성능이 기준을 충족하는가?"
- "컨트롤 플레인 병목 없이 확장 가능한가?"

자세한 내용은 [05-CAT-활용.md](./05-CAT-활용.md) 참조.

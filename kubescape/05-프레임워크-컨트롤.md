# 프레임워크 및 컨트롤

Kubescape의 보안 프레임워크 및 컨트롤에 대해 상세히 설명합니다.

## 프레임워크 (Frameworks)

보안 기준을 정의하는 프레임워크들입니다.

### 주요 프레임워크

| 프레임워크 | 설명 | 용도 |
|-----------|------|------|
| **NSA-CISA** | NSA-CISA Kubernetes 보안 가이드 | 미국 정부 기준 준수 검증 |
| **CIS** | CIS Kubernetes Benchmark | 산업 표준 보안 설정 검증 |
| **MITRE ATT&CK** | MITRE ATT&CK 프레임워크 | 공격 벡터 기반 검증 |
| **Custom** | 사용자 정의 프레임워크 | 조직별 보안 기준 정의 |

### 프레임워크 사용 예시

```bash
# NSA-CISA 프레임워크 스캔
kubescape scan framework nsa

# CIS Benchmark 스캔
kubescape scan framework cis

# 모든 프레임워크 스캔
kubescape scan framework all
```

## 컨트롤 (Controls)

개별 보안 검사 항목들입니다.

### 주요 컨트롤 예시

| 컨트롤 ID | 설명 | 검사 항목 |
|----------|------|----------|
| **C-0018** | Readiness probe 정의 여부 | 모든 워크로드에 readiness probe 정의 |
| **C-0003** | Privileged 컨테이너 사용 금지 | Privileged 컨테이너 검사 |
| **C-0052** | 이미지 취약점 검사 | Critical/High 취약점 검사 |

### 컨트롤 사용 예시

```bash
# 단일 컨트롤 스캔
kubescape scan control C-0018

# 여러 컨트롤 스캔
kubescape scan control C-0018,C-0003,C-0052
```

## 사용자 정의 프레임워크

조직별 보안 기준을 정의할 수 있습니다.

```yaml
# custom-framework.yaml 예시
apiVersion: v1
kind: ConfigMap
metadata:
  name: custom-framework
data:
  framework.yaml: |
    frameworks:
      - name: custom-security
        controls:
          - name: C-0018
            description: "Readiness probe must be defined"
```

# 실무 체크리스트

Testkube를 실무에 적용할 때 확인해야 할 사항들을 정리합니다.

## 실행 전 체크리스트

### 1. 환경 준비

| 항목 | 확인 사항 | 확인 방법 |
|------|----------|----------|
| **Kubernetes 버전** | Kubernetes 1.17 이상 지원 여부 확인 | `kubectl version --short` |
| **Testkube 설치** | Testkube 설치 여부 확인 | `testkube status` |
| **CRD 확인** | Test, TestWorkflow CRD 확인 | `kubectl get crd | grep testkube` |
| **권한** | Kubernetes 리소스 조작 권한 확인 | `kubectl auth can-i create test --all-namespaces` |
| **리소스** | 테스트 실행을 위한 충분한 리소스 확인 | `kubectl top nodes` |

### 2. 테스트 준비

| 항목 | 확인 사항 | 비고 |
|------|----------|------|
| **테스트 타입** | Postman, k6, Cypress 등 테스트 타입 확인 | 필요한 Executor 확인 |
| **테스트 소스** | Git, 파일, 문자열 등 테스트 소스 확인 | 테스트 정의 방법 확인 |
| **TestWorkflow** | 복합 테스트 워크플로우 구성 여부 확인 | 여러 테스트 조합 확인 |

## 실행 중 체크리스트

### 1. 상태 모니터링

| 항목 | 확인 방법 | 비고 |
|------|----------|------|
| **테스트 상태** | `testkube get execution <id>` | 정기적으로 확인 |
| **Pod 상태** | `kubectl get pods -l testkube.io/test=<test-name>` | 테스트 실행 Pod 확인 |
| **로그 확인** | `testkube logs <execution-id>` | 테스트 실행 로그 확인 |

## 실행 후 체크리스트

### 1. 결과 확인

| 항목 | 확인 방법 | 비고 |
|------|----------|------|
| **테스트 결과** | `testkube get execution <id> --output json` | status: passed 확인 |
| **아티팩트** | `testkube download artifacts <id> --output-dir ./artifacts` | 테스트 아티팩트 확인 |
| **합격 여부** | 테스트 실행 성공 여부 확인 | 합격 기준 충족 여부 |

### 2. 리포트 생성

| 산출물 | 생성 방법 | 비고 |
|--------|----------|------|
| **TestExecution CRD** | `kubectl get testexecution <id> -o yaml` | 테스트 실행 결과 원본 |
| **로그** | `testkube logs <execution-id>` | 테스트 실행 로그 |
| **아티팩트** | `testkube download artifacts <id> --output-dir ./artifacts` | 테스트 아티팩트 |

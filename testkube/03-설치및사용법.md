# Testkube 설치 및 사용법

Testkube를 설치하고 실행하여 결과를 확인하는 전체 워크플로우를 설명합니다.

## 설치

### CLI 설치

```bash
# Linux/macOS
curl -s https://raw.githubusercontent.com/kubeshop/testkube/main/scripts/install-cli.sh | bash

# 설치 확인
testkube version
```

### Testkube 설치 (클러스터에)

```bash
# Testkube 설치
testkube install

# 설치 확인
testkube status
```

## 기본 워크플로우

### 1. 테스트 생성

```bash
# Postman 테스트 생성
testkube create test --name my-api-test --type postman/collection --git-uri https://github.com/example/postman-collection.git

# k6 성능 테스트 생성
testkube create test --name my-perf-test --type k6/script --git-uri https://github.com/example/k6-script.js

# Cypress E2E 테스트 생성
testkube create test --name my-e2e-test --type cypress/project --git-uri https://github.com/example/cypress-project.git
```

### 2. 테스트 실행

```bash
# 테스트 실행
testkube run test my-api-test

# 실행 상태 확인
testkube get execution my-api-test

# 실행 결과 확인
testkube get execution my-api-test --output json
```

### 3. TestWorkflow 생성 및 실행

```bash
# TestWorkflow YAML 파일 생성
cat > workflow.yaml <<EOF
apiVersion: tests.testkube.io/v1
kind: TestWorkflow
metadata:
  name: my-workflow
spec:
  steps:
    - name: api-test
      test: my-api-test
    - name: perf-test
      test: my-perf-test
EOF

# TestWorkflow 생성
testkube create testworkflow --file workflow.yaml

# TestWorkflow 실행
testkube run testworkflow my-workflow
```

### 4. 결과 확인

```bash
# 테스트 결과 확인
testkube get test my-api-test

# 실행 로그 확인
testkube logs execution <execution-id>

# 아티팩트 다운로드
testkube download artifacts <execution-id> --output-dir ./artifacts
```

# ClusterLoader2 리포트 확인 방법

## 생성되는 리포트 파일

ClusterLoader2는 `--report-dir`로 지정한 디렉토리에 다음 파일들을 생성합니다:

| 파일 | 설명 | 확인 방법 |
|------|------|----------|
| `junit.xml` | JUnit 형식 테스트 결과 리포트 | `cat reports/junit.xml` |
| `generatedConfig_*.yaml` | 실행 시 실제 사용된 설정 파일 | `cat reports/generatedConfig_*.yaml` |
| `cl2-metadata.json` | 메타데이터 | `cat reports/cl2-metadata.json` |

## junit.xml 리포트 확인

### 1. 기본 확인

```bash
# 리포트 파일 확인
cat reports/junit.xml

# 또는 XML 포맷팅하여 보기
cat reports/junit.xml | xmllint --format -
```

### 2. junit.xml 구조

```xml
<testsuite name="ClusterLoaderV2" tests="0" failures="2" errors="0" time="10.05">
    <testcase name="simple-pod-test overall" classname="ClusterLoaderV2" time="10.046">
        <failure type="Failure">[오류 내용]</failure>
    </testcase>
    <testcase name="simple-pod-test: [step: 01] create-pods" ...>
        ...
    </testcase>
</testsuite>
```

**항목 설명:**
- `tests`: 전체 테스트 수
- `failures`: 실패한 테스트 수
- `errors`: 에러 수
- `time`: 실행 시간 (초)
- `<testcase>`: 각 테스트 케이스 (step별)
- `<failure>`: 실패한 경우 오류 내용

### 3. 성공 시 예상 내용

테스트가 성공하면 measurement 결과가 포함됩니다:

```xml
<testsuite name="ClusterLoaderV2" tests="1" failures="0" ...>
    <testcase name="PodStartupLatency" classname="ClusterLoaderV2" time="5.2">
        <properties>
            <property name="p50" value="1.2s"/>
            <property name="p90" value="2.5s"/>
            <property name="p99" value="5.0s"/>
        </properties>
    </testcase>
</testsuite>
```

## kube-burner와의 차이점

### ClusterLoader2
- ✅ **junit.xml 자동 생성** (CI/CD 통합 용이)
- ✅ **표준 JUnit 형식** (대부분 CI/CD 시스템 지원)
- ✅ **Measurement 결과 포함** (성공 시 p50, p90, p99 등)

### kube-burner
- ❌ **junit.xml 없음**
- ✅ `reports/metrics.json` (Prometheus 있을 때만)
- ✅ `reports/thresholds.json` (Prometheus 있을 때만)

## 실제 리포트 확인 예시

```bash
cd clusterloader2/test_poc

# junit.xml 확인
cat reports/junit.xml

# 생성된 설정 확인
cat reports/generatedConfig_*.yaml

# 메타데이터 확인
cat reports/cl2-metadata.json
```
